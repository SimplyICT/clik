Design: Database Schema

New Tables

assets_v2

Same core fields as current (name, code, category, subCategory, status, criticality, manufacturer, model, serial#, customerId, locationId, dates, notes) plus:

  • qr_code — unique identifier string
  • photo_urls — text[] (array of Supabase Storage URLs)
  • assigned_contractor_id — FK to contractors
  • custom_fields — JSONB (category-specific fields)
  • lifecycle_status — active/retired/moved/transferred
  • parent_asset_id — nullable FK, for component relationships

asset_parts

Inventory items used with assets

  • id, asset_id (nullable), name, sku, quantity, min_threshold, unit
  • Contractor records "used 2 filters" against a job

asset_part_usage

Tracks parts consumed during jobs

  • id, part_id, request_id, quantity, used_by, used_at

Existing Table Changes

requests — add nullable asset_id FK to assets_v2

assets — left untouched, deprecated
