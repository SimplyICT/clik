How do jobs relate to the existing request system?

We already have a request lifecycle (11 statuses).

A

Same System, Asset-Enabled

Just add an "asset_id" to the existing request. A job IS a request, but linked to an asset. Reuses all existing request workflows.

B

New Job System

Jobs are separate from requests. A job has its own lifecycle: Created → Assigned → In Progress → Completed → Verified. Job and request can be linked but are distinct.

C

Same Statuses, Job Terminology

Reuse the request state machine but call it "job" in the UI. Link assets to requests via a new field. Keeps backend simpler.
