Standalone Service — what does that mean here?

Current server is one FastAPI file. Two options:

A

New Module in Same Server

A dedicated server/asset_service/ directory with its own router, DB tables, and API prefix (/api/asset-management/*). Lives in the same FastAPI process but is self-contained. Shares auth middleware and DB connection.

B

Separate Microservice

Entirely separate Python service on its own port. Talks to a different Supabase project or schema. Communicates with main server via HTTP or message queue.
